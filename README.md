SIP - сервис подписок и регистраций
=====

# Функциональные возможности

Данное приложение реализует SIP-сервер на базе Erlang/OTP и библиотеки NkSIP. Приложение использует базу данных Mnesia для хранения абонентов, подписок и регистраций. 

### Аутентификация

Проверка пароля пользователя в базе данных в таблице abonent.

### Регистрация

Обработка REGISTER, сохранение контакта, времени истечения регистрации, генерация событий при регистрации, разрегистрации и запросе активных регистраций пользователя. Поддерживается несколько контактов одного абонента.

### Подписка

Обработка SUBSCRIBE, хранение подписок, отправка NOTIFY при изменении статуса, возможность отписаться. При разрегистрации пользователя его подписке сохраняются в базе, но уведомления по ним не отправляются, пока пользователь не зарегистрируется снова.

### Хранение данных

Все абоненты, активные подписки и регистрации хранятся в базе данных Mnesia.

### Очистка истекших подписок и регистраций

Каждые 15 секунд (время по умолчанию) из базы данных удаляются подписки и регистрации с истекшим временем действия.

### Мониторинг статистики

Вывод статистики: текущего времени, кол-во подписок и кол-во регистраций в файл active.log

### Добавление и удаление абонентов

Функции модуля sip_serv_auth позволяют добавлять и удалять пользователя через терминал. Для добавления необходимо указать AOR и пароль пользователя, удаление происходит по AOR.

# Установка и запуск проекта

### Клонирование репозитория

```
git clone https://study-git.eltex-co.ru/erlang-projects/2026-july-group/sip-subscription-service.git
```

Git автоматически создаст папку с именем проекта и скачает все файлы с репозитория. Для начала работы необходимо перейти в папку проекта и открыть в ней терминал.

### Требования к окружению

Проект предназначен для запуска в Docker-контейнере. Для работы необходим Docker, все необходимые зависимости уже включены в образ.

### Сборка образа

```
docker build -t sip-serv .
```

### Запуск контейнера

Рекомендуется использовать сетевой режим host чтобы контейнер мог принимать SIP-запросы напрямую на IP-адрес хоста
```
docker run -it --rm --name sip-serv --network host -v "$(pwd)/logs:/app/logs" sip-serv
```
Для работы в изолированной сети можно использовать проброс портов
```
docker run -it --rm --name sip-serv -p 5060:5060/udp -v "$(pwd)/logs:/app/logs" sip-serv
```
В обоих случаях после запуска вы попадете в интерактивную оболочку Erlang с запущенным приложением sip_serv, все логи будут выводиться в консоль. Также локально будет создана директория logs, в которую при выполнении команды sip_serv_monitor:log_stats_to_file() будет помещен файл active.log

### Остановка и удаление контейнера

Для остановки работающего приложения используйте сочетание Ctrl+G, затем нажмите q для выхода.
Для остановки работы контейнера можно использовать сочетание Ctrl+C или команду
```
docker stop sip-serv
```
При необходимости можно удалить образ.
```
docker rmi sip-serv
```

# Архитектура проекта

Проект состоит из следующих модулей:

1. sip_serv_app - корневое приложение: создает схему Mnesia, таблицы, загружает абонентов из sys.config, запускает супервизор
2. sip_serv_sup - супервизор, запускает процесс sip_serv_cleanup и NkSIP обработчик (sip_serv_handler)
3. sip_serv_cleanup - модуль, реализующий поведение gen_server, отвечающий за очистку истекших регистраций и подписок, вызывает функцию очистки бд каждые 15 секунд
4. sip_serv_db - модуль управлением базой данных Mnesia, отвечает за создание, удаление и поиск регистраций, подписок и абонентов
5. sip_serv_auth - модуль, отвечающий за работу с абонентами
6. sip_serv_handler - модуль-обработчик NkSIP, принимает SIP запросы, обрабатывает REGISTATION и SUBSCRIBE
7. sip_serv_monitor - модуль сбора статистики и ее вывода в файл active.log
8. sip_serv_conf - модуль для чтения файла конфигурации, который содержит список абонентов
9. sip_serv_records - заголовочный файл, содержащий записи для формирования таблиц

Таблицы Mnesia:
1. abonent - хранит данные об абонентах (тип - set, ключ - aor)
2. registration - хранит данные об активных регистрациях (тип - bag, ключ - aor)
3. erlsubscription - хранит данные об активных подписках (тип - set, ключ - id)
4. publication - хранит данные о состояниях полбзователя publish (тип - bag, ключ - aor)

# Конфигурация

Файл sys.config содержит:

1. Путь к данным Mnesia - директория /tmp/mnesia (создается при сборке образа)
2. Список абонентов - каждая запись содержит AOR, пароль и домен

# Документация

Исходный код имеет документацию, написанную с помощью edoc. Для ее генерации необходимо собрать образ и выполнить команду
```
docker run -it --rm --name sip-serv --network host -v "$(pwd)/doc:/app/doc" sip-serv sh -c "rebar3 edoc"
```
Документация появится в каталоге doc/, которая может быть открыта только от root. Для выдачи прав на открытие файлов в фапае необходимо выполнить команды
```
sudo chown -R $USER:$USER doc
chmod -R u+rwX doc
```
Для проссотра необходимо открыть файл index.html в браузере.

# Проверка кода с Dialyzer

Для анализа типов используется Dialyzer. В Docker проверку можно выполнить с помощью команды
```
docker run --rm -it sip-serv rebar3 dialyzer
```
Для работы с Dialyzer каждая функция проекта содержит спецификацию.

# Выполнение команд в рантайме

При запуске приложения в консоли можно выполнить слудующие команды:

- sip_serv_monitor:log_stats_to_file() - запись статистики (количество активных регистраций и подписок) в файл logs/active.log
- sip_serv_auth:add_abonent(AOR, Password) - добавление нового пользователя, в функцию передается его уникальный идентификатор и пароль
- sip_serv_auth:delete_abonent(AOR) - удаление пользователя по его уникальному идентификатору

# Тестирование

## Twinkle

1. Установить Twinkle  
    sudo apt-get install twinkle
2. Перенести файлы из /test/twinkle в папку ~/.twinkle  
    cp -r /test/twinkle/* ~/.twinkle
3. Запустить Twinkle, выбрать необходимых клиентов, произвести регистрацию. В логах контейнера появится сообщение об успешной регистрации. Произвести ручную разрегистрацию или подождать ее истечения, появятся логи о разрегистрации. Также можно запросить список активных регистраций, будут выведены все регистрации клиента и оставшееся время их длительности.
4. Для проверки подписки необходимо зарегистрировать несколько пользователей и добавить их в друзья.

## SIPP

Скрипты для тестирования находятся в папке /test/sipp. В ней находятся сами SIPP-скрипты и скрипт на языке bash для их запуска.

### Установка SIPP

sudo apt-get install sipp

### Необходимо сделать файлы исполняемыми

Находясь в папке /test/sipp нужно открыть терминал и выполнить команду
```
chmod +x run_test.sh
```

### Выполнение скриптов

Необходимо запустить скрипт
```
./run_test.sh
```
Будут зарегистрированны два пользователя user1 и user2, далее user1 подпишется на user2, будет отправлен NOTIFY и скрипт завершится. В логах выполнения скрипта будут выведены вспомогательные сообщения о регистрации пользователей, подписке и успешном завершении выполнения тестов. В логах сервера можно увидеть работу SIP-сервера.
